## What to Build
A currency conversion cache backed by PostgreSQL. The `FXRateRefreshWorkflow` (Temporal) fetches rates from exchangerate-api.com every 6 hours and upserts them into `fx_rates`. The `convert()` service method is called synchronously by `budgets` and `investments` — it never makes a live API call at request time.

INR is the base currency: all rates are stored as X→INR. Non-INR pairs (USD→EUR) are triangulated through INR at query time.

## Tables to Create
| Table | Key columns |
|---|---|
| `fx_rates` | `from_currency` CHAR(3), `to_currency` CHAR(3), `rate` NUMERIC(18,6), `fetched_at` TIMESTAMPTZ |

**Unique constraint**: `UNIQUE (from_currency, to_currency)` — upserted on every refresh.

No `outbox` table — `fx` publishes no domain events.

## Service Methods Exposed (cross-domain — Pattern 3)
### `convert(amount, from_currency, to_currency, as_of_date?) → Decimal`
```python
def convert(
    amount: Decimal,
    from_currency: str,
    to_currency: str,
    as_of_date: date | None = None,
) -> Decimal
```
- Returns `amount` unchanged if `from_currency == to_currency`
- For non-INR pairs: triangulate via INR (`amount_eur = amount_usd * (USD→INR) / (EUR→INR)`)
- If no rate found: raises `FXRateUnavailableError` with currencies and last rate timestamp
- If rate is >24 hours old and `as_of_date` is None: log a warning (do not raise)

## Action Steps

### Step 1 — Create `models.py`
Define `FXRate` model.
- Inherit `Base`, `IDMixin`, `TimestampMixin` (no `updated_at` — immutable log rows, upserted not updated)
- `from_currency` and `to_currency`: `CHAR(3)`, not null
- `rate`: `NUMERIC(18, 6)`, not null
- `fetched_at`: `TIMESTAMPTZ`, not null
- Unique constraint on `(from_currency, to_currency)`

### Step 2 — Create Alembic migration
`uv run alembic revision --autogenerate -m "fx: add fx_rates table"`.
Confirm the unique constraint and numeric precision appear correctly in the generated SQL.

### Step 3 — Create `repositories.py`
Key methods:
- `get_rate(from_currency, to_currency) -> FXRate | None` — latest row for the pair
- `get_rate_as_of(from_currency, to_currency, as_of_date) -> FXRate | None` — closest rate at or before `as_of_date`
- `upsert_rate(from_currency, to_currency, rate, fetched_at) -> None` — `INSERT ... ON CONFLICT (from_currency, to_currency) DO UPDATE SET rate = ..., fetched_at = ...`
- `list_all_rates() -> list[FXRate]` — all current rates (for the API endpoint)
- `get_all_non_inr_currencies() -> list[str]` — used by the workflow to know which pairs to fetch

### Step 4 — Create `services.py`
- `convert(amount, from_currency, to_currency, as_of_date?, session) -> Decimal`
  - Implements triangulation and staleness warning as described above
- `refresh_rates(currencies_to_fetch, exchangerate_client, session) -> None`
  - Called by the Temporal activity; upserts each fetched rate via the repo

### Step 5 — Create `schemas.py`
- `FXRateResponse` — from_currency, to_currency, rate, fetched_at
- `ConvertResponse` — from_currency, to_currency, original_amount, converted_amount, rate_used, fetched_at

### Step 6 — Create `events.py`
No events published or subscribed. File contains only a module docstring explaining that `fx` uses no outbox.
```python
# fx domain publishes no events and subscribes to none.
# FX rate refreshes are driven by the FXRateRefreshWorkflow Temporal schedule.
```

### Step 7 — Create `workflows/fx_rate_refresh.py`
Temporal workflow scheduled every 6 hours:
1. Activity: call `exchangerate-api.com` for all currencies required (`platform/clients/exchangerate.py`)
2. Activity: collect non-INR currencies from `bank_accounts`, `credit_cards`, and `instruments` tables (plus fixed set: USD, EUR, GBP, SGD, AED)
3. Activity: upsert results into `fx_rates` via `FXService.refresh_rates()`

Workflow must be deterministic — all side effects (API calls, DB writes) in activities only.

### Step 8 — Complete `api.py`
2 endpoints. The `GET /fx/convert` endpoint validates `from` and `to` are valid ISO 4217 codes (3 uppercase letters). Returns 503 if `FXRateUnavailableError` is raised.
